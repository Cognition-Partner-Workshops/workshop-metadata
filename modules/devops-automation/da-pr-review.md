# E3 — PR Review

## Challenge

Use Devin to assist in reviewing a Pull Request. Optionally have Devin contribute to the PR by adding a commit.

## Options

- **Repository:** Any repo in the org
- **Prerequisite:** An open PR to review (can be created by another participant, by Devin in a previous challenge, or a pre-staged PR)

## Task

Use Devin to assist you in reviewing a Pull Request. Optionally have Devin contribute to the PR by adding a commit to improve the PR's branch.

## Suggested Approaches

1. **Code Review:** Ask Devin to review a PR for correctness, style, security, and performance
2. **Fix a PR:** Ask Devin to add a commit to an existing PR to fix an issue found during review
3. **Enhance a PR:** Ask Devin to add tests, documentation, or error handling to a PR

## Sample Prompt

> Review PR #[number] in [repo]. Check for: code correctness, security issues, test coverage gaps, and adherence to the project's coding conventions. Provide a detailed review comment. If you find issues, push a fix commit to the PR branch.

## What Participants Will Learn

- How Devin reads and interprets PR diffs
- Devin's code review capabilities (what it catches vs. misses)
- How to use Devin as a reviewer + contributor in the same session
- How Devin comments on PRs (inline comments, general comments)

## Devin Features Exercised

- PR review and commenting
- Code quality assessment
- Optional: commit to existing PR branch
- GitHub integration

## Difficulty

Beginner to Intermediate

## Estimated Time

30 minutes

## Notes

- This is a great low-risk challenge — reviewing code doesn't change anything
- Works well as a paired activity: one participant creates a PR, another uses Devin to review it
- Tip: Try giving Devin feedback on its PR review to see how it adjusts
