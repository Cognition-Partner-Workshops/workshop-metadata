# A2 — Unit Testing

## Challenge

Add or improve unit test coverage for a repository, create a test coverage report, and remediate any failing tests.

## Options

- **Repository:** Any repo in the org (participant's choice)
- **Recommended starting points:**
  - [app_timesheet-client](https://github.com/Cognition-Partner-Workshops/app_timesheet-client) — Jest tests exist in `backend/src/__tests__/`
  - [ts-java-spring-boot-realworld-example-app](https://github.com/Cognition-Partner-Workshops/ts-java-spring-boot-realworld-example-app) — JUnit tests exist
  - [app_petclinic-backend](https://github.com/Cognition-Partner-Workshops/app_petclinic-backend) — Spring Boot test suite

## Task

1. Add a unit testing framework if the repo does not have unit testing defined
2. Improve test coverage and create a test coverage report
3. Remediate failing tests

## Sample Prompt

> Analyze the current test coverage of [repo]. Add missing unit tests to increase coverage to at least 80%. Generate a coverage report and fix any failing tests. Open a PR with the changes.

## What Participants Will Learn

- How Devin understands existing test patterns and extends them
- How Devin selects appropriate testing frameworks for a tech stack
- Devin's ability to generate meaningful (not trivial) test cases
- How to evaluate AI-generated tests for quality

## Devin Features Exercised

- Codebase analysis and understanding
- Test generation
- PR creation with coverage reports

## Difficulty

Beginner to Intermediate — depends on the chosen repository.

## Estimated Time

45 minutes
