# A3 — End-to-End Testing

## Challenge

Experiment with Devin's capabilities for end-to-end testing against a locally hosted application.

## Options

- **Repository:** [cal.com](https://github.com/Cognition-Partner-Workshops/cal.com)
- **Alternative:** [app_timesheet-client](https://github.com/Cognition-Partner-Workshops/app_timesheet-client) (simpler app, good for first-timers)

## Task

Experiment with Devin's capabilities for end-to-end testing on the locally hosted application. This is intentionally open-ended — explore what Devin can do with browser-based testing.

## Suggested Approaches

- Have Devin write Playwright E2E tests for cal.com (existing Playwright config available)
- Have Devin use its built-in browser to manually test workflows and report bugs
- Have Devin record a screen recording of its E2E test execution

## Prerequisites

- **Runtime required:** The application must be running locally on Devin's machine or hosted externally
- **cal.com:** `yarn dev` starts on port 3000
- **app_timesheet-client:** Backend on port 3001, frontend on port 5173

## What Participants Will Learn

- How Devin interacts with running applications via its browser
- Devin's screen recording capabilities for test evidence
- Writing and running Playwright/Cypress tests with Devin

## Devin Features Exercised

- Browser interaction
- Screen recording
- Test framework setup and execution
- Runtime application testing

## Difficulty

Intermediate — requires understanding of E2E testing concepts.

## Estimated Time

60 minutes
